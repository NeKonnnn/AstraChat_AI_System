# Установка vLLM - Решение проблем

## Проблема: vLLM не найден при запуске приложения

Если вы видите ошибку "vLLM is not available", это означает, что vLLM установлен в другом виртуальном окружении, чем то, которое использует приложение.

## Решение

### 1. Определите, какое окружение использует приложение

При запуске приложения вы увидите информацию о Python окружении:
```
🐍 PYTHON ENVIRONMENT INFO
Python executable: путь/к/python.exe
```

### 2. Установите vLLM в правильное окружение

#### Вариант A: Если приложение использует venv_312

```bash
# Активируйте виртуальное окружение
.\venv_312\Scripts\activate  # Windows
# или
source venv_312/bin/activate  # Linux/Mac

# Установите vLLM
pip install vllm
```

#### Вариант B: Если приложение использует другое окружение

```bash
# Активируйте виртуальное окружение, которое использует приложение
# (путь указан в выводе "Python executable")

# Установите vLLM
pip install vllm
```

#### Вариант C: Установка в конкретное окружение напрямую

```bash
# Замените путь на путь к Python из вывода приложения
.\venv_312\Scripts\python.exe -m pip install vllm
```

### 3. Проверьте установку

```bash
# Используйте тот же Python, что и приложение
.\venv_312\Scripts\python.exe -c "import vllm; print('vLLM version:', vllm.__version__)"
```

Должно вывести версию vLLM (например: `vLLM version: 0.13.0`)

### 4. Перезапустите приложение

После установки перезапустите приложение. Теперь вы должны увидеть:
```
✅ vLLM package found (version: 0.13.0)
✅ Selected backend: vLLM
```

## Проблема: "No module named 'vllm._C'"

Если вы видите ошибку `No module named 'vllm._C'`, это означает, что vLLM установлен, но C++ расширения не скомпилированы.

### Причины:
1. **Windows**: vLLM требует Visual Studio Build Tools для компиляции C++ кода
2. **Неполная установка**: vLLM установился без компиляции расширений
3. **Отсутствие CUDA**: vLLM требует CUDA Toolkit для правильной компиляции

### Решения:

#### Вариант 1: Компиляция vLLM на Windows

**⚠️ ВАЖНО:** vLLM официально не поддерживает Windows. Компиляция сложная и может не работать.

**Подробная инструкция:** См. файл `VLLM_WINDOWS_BUILD.md`

**Краткая версия:**
1. Установите Visual Studio Build Tools 2022
2. Установите CUDA Toolkit 12.x
3. Используйте Python 3.10 или 3.11 (не 3.12+)
4. Установите vLLM из исходников:
   ```bash
   pip install git+https://github.com/vllm-project/vllm.git
   ```

**Время компиляции:** 30-60 минут
**Сложность:** Высокая

#### Вариант 2: Использовать llama.cpp (рекомендуется для Windows)

Llama.cpp работает на Windows без дополнительных инструментов и поддерживает как CPU, так и GPU:

В `config/config.yml`:
```yaml
model:
  backend: "llama.cpp"  # вместо "vllm"
```

#### Вариант 3: Использовать Docker с предустановленным vLLM

Если нужен именно vLLM, используйте официальный Docker образ:
```bash
docker pull vllm/vllm-openai:latest
```

## Важные замечания

1. **vLLM требует GPU (CUDA)** - на CPU не работает
2. **Windows проблемы** - vLLM сложно установить на Windows из-за необходимости компиляции C++ кода
3. **Конфликты зависимостей** - если видите предупреждения о langchain, это нормально, vLLM все равно должен работать
4. **Версия Python** - vLLM требует Python 3.8+
5. **Рекомендация для Windows** - используйте llama.cpp, он проще в установке и работает отлично

## Альтернатива: Использование llama.cpp

Если у вас нет GPU или возникают проблемы с vLLM, используйте llama.cpp:

В `config/config.yml`:
```yaml
model:
  backend: "llama.cpp"  # вместо "vllm"
```

Llama.cpp работает на CPU и GPU, и не требует CUDA.
